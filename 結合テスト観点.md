# 結合テスト観点

> バージョン: 1 | 更新日時: 2026/6/24 15:40:55

## テスト観点ツリー

### ITV-L1-001 DB結合

#### ITV-L2-001 SQL結合・検索および更新 (イベント・プロジェクト管理API群)

- **ITV-L3-001 データ整合性検証** [高]
  - eventsテーブルへのレコード挿入時、start_timeおよびend_timeの整合性が保たれていること
  - projectsテーブルのstatus更新が、設計されたステータス遷移（準備中→発表中→終了）通りに行われること
  - questionsテーブルに投稿されたレコードが、承認ステータス更新後に適切に反映されること

#### ITV-L2-002 トランザクション制御 (Q&A管理および投票管理)

- **ITV-L3-002 更新時の排他・整合性** [高]
  - votesテーブルへの投票時、制限回数チェックと同時にトランザクション内で整合性が担保されること
  - 質問投稿時、バリデーションエラー発生時にデータベースの状態が未更新（ロールバック）であること
  - プロジェクト更新時に楽観ロック等で競合が制御されていること

#### ITV-L2-004 SQL結合・検索 (イベント・プロジェクト管理 API群)

- **ITV-L3-004 イベントデータおよびプロジェクトデータの整合性検証** [高]
  - eventsテーブルのデータがイベント作成後のGETイベント一覧取得で正しく参照されること
  - projectsテーブルのstatus更新がプロジェクト更新APIにより正しく反映されること
  - GET /api/projectsによるプロジェクト一覧取得時に、対応するイベント情報とJOINして取得できること

#### ITV-L2-005 トランザクション制御 (Q&A管理・投票管理 API群)

- **ITV-L3-005 ステータス更新の排他および一貫性** [高]
  - 質問ステータス更新時に、questionsテーブルのステータスが期待通り更新されロールバック不要であること
  - 複数管理者による同一質問への同時ステータス更新時の整合性保護
  - トランザクション終了時に正しいコミット単位で反映されること

#### ITV-L2-008 トランザクション制御 (POST /api/votes)

- **ITV-L3-008 投票制限の整合性** [高]
  - votesテーブルへのレコード挿入時に、1人あたりの投票数制限が正しく適用されること
  - 制限回数を超えた場合、トランザクションがロールバックされE004が返却されること
  - 同時投票発生時、排他制御により制限数を超えた投票が許可されないこと

#### ITV-L2-009 SQL結合・検索 (GET /api/admin/events/{event_id}/export)

- **ITV-L3-009 集計データの抽出検証** [中]
  - questionsテーブルとvotesテーブルをevent_idで結合し、対象イベントの全データが抽出されること
  - イベント期間外やデータが存在しない場合、空のCSVが出力されること
  - 集計結果の件数および内容が仕様通りに抽出されること

### ITV-L1-002 API結合

#### ITV-L2-003 ステータスコード・例外処理 (API群)

- **ITV-L3-003 ビジネスルール違反の正確性** [高]
  - 終了日時が開始日時より前のPOST /api/events時にE001エラーが返却されること
  - 投票制限数に達した状態でPOST /api/votesを呼び出した際にE004エラーが返却されること
  - 存在しないプロジェクトIDに対するPATCH /api/projects/{id}時にE002エラーが返却されること

#### ITV-L2-006 HTTPメソッド別動作とステータスコード (全API対象)

- **ITV-L3-006 正常系および異常系ステータスの検証** [高]
  - POSTイベント作成時に正しく201が返却されること
  - 必須パラメータ欠落時に400等の適切なバリデーションエラーが返却されること
  - 未認証または権限不足時の401/403エラー応答の正確性

#### ITV-L2-010 ステータスコード (POST /api/admin/system/monitor)

- **ITV-L3-010 システム権限による実行検証** [中]
  - system権限以外のユーザーがアクセスした場合に403エラーとなること
  - イベント状態の評価が正常終了した場合に200が返却されること
  - 異常パラメータ指定時に適切なエラーハンドリングがなされること

#### ITV-L2-012 認証・認可制御 (全APIエンドポイント)

- **ITV-L3-012 認証方式の検証** [高]
  - Bearer Tokenを含まないリクエストに対して401 Unauthorizedが返却されること
  - 無効なフォーマットのトークンを付与した場合に401 Unauthorizedが返却されること
  - 有効期限切れのトークンを使用した場合に401 Unauthorizedが返却されること
- **ITV-L3-013 認可（パーミッション）の検証** [高]
  - user権限がadmin用APIを呼び出した場合に403 Forbiddenが返却されること
  - system権限以外のユーザーがシステム管理用APIを呼び出した場合に403 Forbiddenが返却されること
  - 特定のイベントIDに対して権限外のユーザーが操作を試みた場合に403 Forbiddenが返却されること

#### ITV-L2-013 共通エラーハンドリング (全APIエンドポイント)

- **ITV-L3-014 異常系レスポンスの検証** [中]
  - DB接続エラー等の予期せぬサーバエラー発生時に500 Internal Server Errorが返却されること
  - リクエストタイムアウト発生時に適切なエラーレスポンスが返却されること
  - レート制限（Rate Limit）を超過したリクエストに対して429 Too Many Requestsが返却されること

### ITV-L1-005 ファイルI/F

#### ITV-L2-007 帳票出力 (データエクスポートAPI)

- **ITV-L3-007 CSV出力の形式検証** [中]
  - イベント終了後に正しいフォーマットでCSVが集計データとして出力されること
  - イベント期間外のアクセス時に400エラーが返却されること
  - 存在しないイベントID指定時にエラー制御が適切に行われること

#### ITV-L2-011 ファイル出力 (GET /api/admin/events/{event_id}/export)

- **ITV-L3-011 CSV出力フォーマット検証** [高]
  - Q&A内容および投票結果が正しいCSVフォーマットで出力されること
  - イベント終了状態の判定ロジックにより、イベント期間中には400が返却されること
  - データ生成失敗時、適切なエラーコードE005を含むレスポンスが返却されること
