## ユーザー認証

参加者がQRコード経由でアクセスしセッションを開始するフロー

**参加者:** 参加者 (actor)、フロントエンド (system)、APIサーバー (system)、データベース (database)

**メッセージフロー:**
- 参加者 → フロントエンド: QRコード読み込み (URLアクセス)
- フロントエンド → APIサーバー: GET /api/events/{code}/auth
- APIサーバー → データベース: イベント存在確認
  - データベース ← APIサーバー: イベント情報
- APIサーバー → APIサーバー: 一時セッション発行
  - APIサーバー ← フロントエンド: セッションToken返却
  - フロントエンド ← 参加者: イベントホーム表示

```mermaid
sequenceDiagram
    actor participant as 参加者
    participant frontend as フロントエンド
    participant api as APIサーバー
    participant db as データベース
    participant->>frontend: QRコード読み込み (URLアクセス)
    frontend->>api: GET /api/events/{code}/auth
    api->>db: イベント存在確認
    db-->>api: イベント情報
    api->>api: 一時セッション発行
    Note over api,api: ※要確認：デバイス識別ロジック
    api-->>frontend: セッションToken返却
    frontend-->>participant: イベントホーム表示
```

## Q&A管理

参加者の投稿から運営者の承認を経て公開されるまでの流れ

**参加者:** 参加者 (actor)、APIサーバー (system)、運営者 (actor)、データベース (database)

**メッセージフロー:**
- 参加者 → APIサーバー: POST /api/questions (投稿)
- APIサーバー → データベース: 未承認状態で保存
  - データベース ← APIサーバー: 保存完了
- APIサーバー → 運営者: 質問通知 (WebSocket)
- 運営者 → APIサーバー: PATCH /api/questions/{id} (承認)
- APIサーバー → データベース: ステータス更新: 承認済
  - データベース ← APIサーバー: 更新完了
- APIサーバー → 参加者: 質問表示更新 (WebSocket)

```mermaid
sequenceDiagram
    actor participant as 参加者
    participant api as APIサーバー
    actor admin as 運営者
    participant db as データベース
    participant->>api: POST /api/questions (投稿)
    api->>db: 未承認状態で保存
    db-->>api: 保存完了
    api-->>admin: 質問通知 (WebSocket)
    admin->>api: PATCH /api/questions/{id} (承認)
    api->>db: ステータス更新: 承認済
    db-->>api: 更新完了
    api-->>participant: 質問表示更新 (WebSocket)
```

## 投票処理

参加者が投票制限内でプロジェクトに投票を行うフロー

**参加者:** 参加者 (actor)、APIサーバー (system)、データベース (database)

**メッセージフロー:**
- 参加者 → APIサーバー: POST /api/votes (投票)
- APIサーバー → データベース: 投票数カウント確認
  - データベース ← APIサーバー: 現在投票数
- APIサーバー → APIサーバー: 制限回数チェック
- APIサーバー → データベース: 投票データ挿入
  - データベース ← APIサーバー: 成功
  - APIサーバー ← 参加者: 投票完了レスポンス

```mermaid
sequenceDiagram
    actor participant as 参加者
    participant api as APIサーバー
    participant db as データベース
    participant->>api: POST /api/votes (投票)
    api->>db: 投票数カウント確認
    db-->>api: 現在投票数
    api->>api: 制限回数チェック
    Note over api,api: ※要確認：投票ルール
    api->>db: 投票データ挿入
    db-->>api: 成功
    api-->>participant: 投票完了レスポンス
```
